Роль: Senior Backend Developer (TypeScript, Node.js, grammY, Prisma ORM).

Задача: Интегрируй в готовый код Telegram-бота "Morena VPN" платежную систему CryptoBot (API Crypto Pay) для приема платежей в рублях (через СБП) и автоматической выдачи ключей. Также реализуй генерацию QR-кодов для оплаты прямо в чате бота.

Используемые библиотеки:
- axios (для HTTP-запросов к API CryptoBot)
- qrcode (для генерации QR-кодов в Buffer памяти)

Шаг 1. Создай модуль `src/cryptoBotApi.ts`:
Реализуй класс `CryptoBotApi`. Он должен брать токен из process.env.CRYPTO_BOT_TOKEN. Если process.env.CRYPTO_BOT_TESTNET === 'true', базовый URL должен быть 'https://cryptobot.app', иначе 'https://cryptobot.app'.
Внутри класса напиши два метода:
1. `createRubInvoice(amountRub: number, payload: string)`: делает POST-запрос на эндпоинт `/createInvoice`. Параметры запроса: 
   - asset: 'USDT' (валюта, которую получаешь ты)
   - amount: количество в виде строки
   - fiat: 'RUB' (валюта, которую видит и платит клиент)
   - currency_type: 'fiat' (чтобы включился p2p-мост под СБП)
   - payload: скрытая строка с данными ("tariffId:tgUserId:discount")
   - description: 'Оплата безопасного сетевого шлюза Morena VPN'
   Метод должен возвращать объект с `invoice_id`, `status` и `pay_url`.
2. `getInvoiceStatus(invoiceId: number)`: делает GET-запрос на `/getInvoices` с параметром `invoice_ids` и возвращает текущий статус строкой ('active', 'paid', 'expired').

Шаг 2. Обнови обработчики тарифов в `bot.ts`:
Когда пользователь выбирает тариф (1 месяц за 300 руб или 3 месяца за 800 руб):
1. Рассчитай цену с учетом скидки с его бонусного баланса в БД (если на балансе есть рубли, вычти их из стоимости тарифа, но оставь минимум 10 рублей для оплаты счета).
2. Вызови метод `createRubInvoice` из созданного класса.
3. Используя библиотеку `qrcode` (метод `QRCode.toBuffer()`), сгенерируй из полученного `pay_url` изображение QR-кода в виде PNG-буфера (Buffer) прямо в оперативной памяти сервера, без сохранения файла на диск.
4. Отправь пользователю нативное сообщение с этим фото (используй `new InputFile(qrBuffer, 'sbp_qr.png')` из grammY).
5. Под фото добавь Inline-клавиатуру с двумя кнопками:
   - URL-кнопка: "💳 Перейти к оплате СБП" (ведет на `pay_url`)
   - Text-кнопка: "✅ Я оплатил" (с callback_data вида `check_payment:INVOICE_ID`)

Шаг 3. Логика проверки платежа и выдачи ключа:
1. При нажатии на кнопку "✅ Я оплатил", бот должен запросить статус инвойса через `getInvoiceStatus`. Если статус не 'paid', выведи вежливое предупреждение, что банк еще обрабатывает перевод по СБП.
2. Реализуй функцию фонового поллинга (проверки): как только счет создан, запускай интервал проверки (`setInterval`) каждые 7 секунд. Если статус стал 'paid', очищай интервал и вызывай функцию успешной выдачи заказа. Если статус стал 'expired', также очищай интервал.
3. При успешной оплате ('paid'):
   - Распарси строку payload.
   - Спиши использованные бонусы с баланса пользователя в SQLite (через Prisma).
   - Вызови готовый метод интеграции с RoyaltyKey API для генерации нового VPN-ключа (или продления старого, если в payload был флаг "renew").
   - Запиши подписку в БД.
   - Отправь пользователю красивое сообщение: "✅ Ваша оплата принята!" и выведи выданный ключ доступа в блоке кода MarkdownV2, чтобы его было удобно скопировать в один клик.

Напиши этот код чисто, с обработкой ошибок (try/catch) и подробными комментариями на русском языке.
